## Priority Philosophy

In order: **PLAN** (anticipate edge cases) → **PHASE 2 FOUNDATION** (~1 day now to save ~2+ weeks later — take it, we're pre-launch) → **FUNCTIONING CODE** (pragmatic, reliable) → **CORRECTNESS** (compiles, type-checks, scoped tests pass deterministically) → **CODE QUALITY** (clean, no warnings) → **POLISH** (docs, formatting).

Non-negotiables: tests drive implementation; satisfy every `qualityDimensions` entry; handle `Option`/`Result` explicitly (no `unwrap()` in production). For `estimatedEffort: "high"` or `modifiesBehavior: true` tasks, note the one alternative you rejected and why. For everything else, pick and go.

**Prohibited outcomes:**

- Tests that only assert 'no crash' or check type without verifying content
- Tests that mirror implementation internals (break when refactoring)
- Abstractions with only one concrete use
- Error messages that don't identify what went wrong or what to do about it

- Hand-written documentation that duplicates clap-discoverable information (defeats Prong 2 drift CI)
- Subprocess invocations of `task-mgr --help` from tests or runtime code (use `clap::CommandFactory` per learning 160)

## Quality Checks

The full test suite is expensive. Per-iteration tasks run a **scoped** gate; **REVIEW-001** runs the full gate and must leave the repo fully green (including pre-existing failures).

### Per-iteration scoped gate (FEAT tasks)

Format → type-check → lint → **scoped tests for touched files** → pre-commit hooks. Fix every failure before committing.

```bash
# Pipe to tee + grep in one command per ~/.claude/CLAUDE.md §3:
cargo fmt --check
cargo check 2>&1 | tee /tmp/check.txt | tail -3 && grep "warning\|error" /tmp/check.txt | head -10
cargo clippy -- -D warnings 2>&1 | tee /tmp/clippy.txt | tail -3 && grep "^error\|^warning" /tmp/clippy.txt | head -10
# Scope test runs - task-mgr is a single crate so use module/fn filters:
cargo test --lib commands::current 2>&1 | tee /tmp/test.txt | tail -5 && grep "FAILED\|error\[" /tmp/test.txt | head -10
# For tests in /tests/ dir, use the test binary name (file stem):
cargo test --test cli_tests 2>&1 | tee /tmp/test.txt | tail -5 && grep "FAILED\|error\[" /tmp/test.txt | head -10
```

Scoping heuristic: start from `touchesFiles`. For each `src/commands/<name>.rs` file, run `cargo test --lib commands::<name>`. For `tests/<name>.rs`, run `cargo test --test <name>`. If you can't determine scope, widen to `cargo test` for the full crate (task-mgr has only one crate).

**Do NOT** run `cargo test` with no filter during regular iterations — that's REVIEW-001's job.

### Full gate (REFACTOR-001 / REVIEW-001)

These tasks run the **full, unscoped** suite on a clean checkout and must finish green:

```bash
cargo fmt --check && cargo check && cargo clippy -- -D warnings && cargo test 2>&1 | tee /tmp/full-gate.txt | tail -10 && grep "FAILED\|error\[" /tmp/full-gate.txt | head -20
```

If ANY test fails — including pre-existing failures that predate this change — REVIEW-001 fixes them. Default: **attempt every failure**, even ones that look out-of-scope. Trunk-green is the invariant this mechanism exists to protect.

Pragmatic escape hatch: if there are **more than ~12 failures AND they're all clearly unrelated to this work**, triage:

1. Fix everything attributable to this change's diff, inline in the REVIEW-001 commit.
2. For the remaining unrelated failures: spawn a single `FIX-xxx` task via `task-mgr add --stdin --depended-on-by REVIEW-001` listing the failing test names + error summaries, and `<promise>BLOCKED</promise>` with that task ID.

Below the ~12-failure threshold, just fix them.

## Data Flow Contracts

These are **verified access patterns** for cross-module data structures in this PRD. Use these exactly — do NOT guess key types from variable names or comments.

### Contract 1: `ResolvedContext` (FEAT-001 → FEAT-004)

**Source struct** (FEAT-001, declared in `src/commands/current.rs`, re-exported from `src/commands/mod.rs`):

```rust
pub struct ResolvedContext {
    pub prefix: String,                  // e.g. "5ba153a7"
    pub source: ResolutionSource,        // enum: EnvVar | SinglePrefix | FromJsonFlag | None
    pub prd_json_path: PathBuf,          // e.g. "tasks/agent-learnability.json"
}

pub enum ResolutionSource {
    EnvVar,         // TASK_MGR_ACTIVE_PREFIX set and matched
    SinglePrefix,   // env unset; DB has exactly one prefix
    FromJsonFlag,   // future: --from-json explicit (placeholder for CLI-fix plan)
    None,           // no PRD resolvable
}
```

**Consumers** (both must use the same struct — no parallel `ResolvedPrefix`/`ActivePrd` etc.):

1. `src/commands/current.rs::run()` — prints the resolved context and exits 0.
2. `src/commands/add.rs::run()` — emits stderr line via shared helper, then proceeds (or refuses if FEAT-004 cross-PRD check trips).

**Access pattern (FEAT-004 cross-PRD check)** — copy-pasteable:

```rust
let resolved: ResolvedContext = resolve_active_context(&conn)?;
let target_prefix = extract_prefix_from_id(&depended_on_by_id);  // e.g. "5ba153a7" from "5ba153a7-MILESTONE-FINAL"
if target_prefix != resolved.prefix {
    let target_prd = lookup_prd_json_for_prefix(&conn, &target_prefix)?;
    return Err(TaskMgrError::CrossPrdReference {
        target_id: depended_on_by_id.clone(),
        target_prd,
        target_prefix,
        active_prefix: resolved.prefix,
        active_prd: resolved.prd_json_path,
    });
}
```

**Key invariant**: `resolved.prefix` is the prefix string (lowercase hex per `src/loop_engine/...`). `target_prefix` extracted from a task id MUST be compared as exact-match (not case-insensitive — see FEAT-001 edge case).

### Contract 2: clap subcommand iteration (FEAT-002)

**Source**: `clap::Command` tree returned by `crate::cli::Cli::command()` (uses `clap::CommandFactory` derive on the root `Cli` struct in `src/cli/mod.rs:32-102`).

**Access pattern** for `src/cli/introspect.rs::generate_command_reference()`:

```rust
use clap::CommandFactory;
let cmd = crate::cli::Cli::command();
for sub in cmd.get_subcommands() {
    if sub.is_hide_set() { continue; }      // exclude hidden

    let about = sub.get_about().map(|s| s.to_string());
    let example = sub.get_after_help().and_then(extract_first_example_line);

    }
}
```

**Key invariant**: `get_subcommands()` returns DIRECT children. Nested subcommands (`loop init`, `loop run`) require one level of recursion. Do NOT recurse deeper — clap top-level has at most 2 levels in this codebase.
