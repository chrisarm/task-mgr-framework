- `iteration_pipeline::ProcessingParams::effective_effort`: `Option<&'static str>` ‚Üí `Option<&'a str>` (borrowed; only feeds `progress::log_iteration`).

**Dispatch sites now prefer plan effort over the static fallback:**
- Sequential `iteration.rs:459`: `base_effort = prompt_result.cluster_effort` (the plan effort); `effort = overflow_override.or(base_effort)`.
- Wave `slot.rs:150`: `effort = slot.effective_effort(override).or(bundle.cluster_effort)`. The redundant static `effort_for_difficulty` call was removed here (the bundle already bakes the static fallback at build time), dropping the now-unused top-level `model` import.
