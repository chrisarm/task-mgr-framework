# Claude Code Agent Instructions

You are an autonomous coding agent implementing **Doctor Orphan-Branch PRD Check** for **task-mgr**.

## Problem Statement

`task-mgr doctor` currently catches stale in-progress tasks, abandoned runs, orphaned relationships, and git-reconciliation gaps — but not a class of pollution that's already happened in production once: a PRD imported into the live `.task-mgr/tasks.db` whose `branch_name` is fake or stale (e.g. a test-fixture branch like `feat/p1-alpha` that never existed in this repo, or a coworker's deleted branch). These orphan PRDs sit in the DB invisibly, get ranked by `task-mgr next` as legitimate work, and silently re-rank above real tasks.

Add a new doctor check that flags any active (non-archived) PRD whose `branch_name` doesn't exist locally or on any remote. Output a remediation hint (`task-mgr archive --branch <branch>`) — but **DO NOT auto-archive on `--fix`**. Doctor's existing auto-fixes are reversible DB cleanups; archiving a PRD whose branch only exists on a coworker's fork would destroy legitimate work. The check is the value — turning silent pollution into a loud warning at the next `doctor` run is enough; the operator runs the suggested archive command explicitly.

## Priority Philosophy

In order: **PLAN** (anticipate edge cases) → **PHASE 2 FOUNDATION** (~1 day now to save ~2+ weeks later — take it, we're pre-launch) → **FUNCTIONING CODE** (pragmatic, reliable) → **CORRECTNESS** (compiles, type-checks, scoped tests pass deterministically) → **CODE QUALITY** (clean, no warnings) → **POLISH** (docs, formatting).

Non-negotiables: tests drive implementation; satisfy every `qualityDimensions` entry; handle `Option`/`Result` explicitly (no `unwrap()` in production). For `estimatedEffort: "high"` or `modifiesBehavior: true` tasks, note the one alternative you rejected and why. For everything else, pick and go.

**Prohibited outcomes:**

- Auto-archiving orphan PRDs on plain `--fix` — must be print-only; archive only via the operator explicitly running `task-mgr archive --branch <branch>`
- Hardcoding `origin/` as the remote prefix — must use generic first-segment strip so `upstream/`, `fork/`, etc. also work
- Two-call branch enumeration (`git branch --list` + `git branch -r`) — one `git for-each-ref refs/heads refs/remotes` call is faster, no `HEAD ->` noise, no enumerate-remotes-by-name fragility
- Erroring when git is unavailable or directory is not a git repo — must return `Ok(vec![])` silently, matching the existing `find_git_reconciliation_tasks` convention
- Flagging PRDs with `branch_name IS NULL` (single-PRD prefix-less layout) or PRDs where every task is already `archived_at IS NOT NULL`
- Tests that only assert 'no crash' or check return type without verifying content
- Tests that mirror implementation internals (break when refactoring)
- Catch-all error handlers that swallow context

## Data Flow Contracts

The new check moves a `(task_prefix: String, project: String, branch_name: String)` tuple from SQL to `doctor()` and into `Issue.entity_id` / `Issue.description` / `Fix.action`. The contract is:

| Layer                                       | Type / Field                                      | Notes                                                              |
| ------------------------------------------- | ------------------------------------------------- | ------------------------------------------------------------------ |
| `prd_metadata` table                        | `task_prefix TEXT UNIQUE`, `project TEXT NOT NULL`, `branch_name TEXT` (nullable) | Schema definition in `src/db/schema/`. Existing rows may have NULL branch_name. |
| `find_orphan_branch_prds` return            | `Vec<(String, String, String)>` — (prefix, project, branch) | Tuple ordering is contractual; tests assert by position. |
| `doctor()` consumer                         | Destructures tuple, builds `Issue { entity_id: prefix.clone(), description: <fmt with project + branch> }` | `entity_id` MUST be the prefix (not project or branch) — matches the convention of other Issue types where entity_id is the primary key. |
| `Fix.action` for print-only                 | `format!("task-mgr archive --branch {}", branch)` | Space-separated CLI form (not `=` form), so users can copy-paste into a shell. |
| JSON serialization (`--format json`)        | `IssueType::OrphanBranchPrd` → `"orphan_branch_prd"` | Automatic via `#[serde(rename_all = "snake_case")]`; do NOT write a manual formatter. |

**Membership test contract** for FEAT-001:

```rust
// Pseudocode for the membership check
fn branch_exists(set: &HashSet<String>, candidate: &str) -> bool {
    if set.contains(candidate) { return true; }
    // Strip first '/'-segment: "origin/feat/foo" → "feat/foo"
    for entry in set {
        if let Some((_, suffix)) = entry.split_once('/') {
            if suffix == candidate { return true; }
        }
    }
    false
}
```

Key behaviors:
- `feat/p1-alpha` candidate, set contains `feat/p1-alpha` → match (direct).
- `feat/foo` candidate, set contains `origin/feat/foo` → match (strip `origin/`).
- `feat/foo` candidate, set contains `upstream/feat/foo` → match (strip `upstream/`).
- `feat/sub/feature` candidate, set contains `origin/feat/sub/feature` → match (split_once strips only the first segment).
- `origin/HEAD -> origin/main` line in raw output → MUST be filtered BEFORE insertion into the set.
