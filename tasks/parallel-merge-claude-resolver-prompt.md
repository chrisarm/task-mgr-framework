# Claude Code Agent Instructions

You are an autonomous coding agent implementing **Parallel slot merge-back: Claude-resolved conflicts** for **task-mgr**.

## Problem Statement

Today, when parallel-slot execution finishes a wave, `worktree::merge_slot_branches` runs `git merge --no-edit <ephemeral>` from slot 0 for each ephemeral slot branch. On any conflict the current code resets slot 0 to its captured pre-merge HEAD, logs a warning, and abandons that slot's commits on the ephemeral branch — losing real work and forcing the user to recover manually.

Two follow-ups, in priority order:

1. **Per-PRD `progress*.txt` conflicts** — already handled (shipped before this PRD): `info/attributes` declares `merge=union` for `tasks/progress*.txt` and `.task-mgr/tasks/progress*.txt`. See `src/loop_engine/worktree.rs::ensure_progress_union_merge` and the unit tests around it. **Do not re-implement.**
2. **Source-code or other content conflicts** — *this PRD's scope*. Spawn Claude in slot 0's already-conflicted worktree, let it resolve, and let the existing fast-forward path bring the resolution back to the ephemeral worktree. The `spawn_claude` infrastructure already exists.

## Priority Philosophy

In order: **PLAN** (anticipate edge cases) → **PHASE 2 FOUNDATION** (~1 day now to save ~2+ weeks later — take it, we're pre-launch) → **FUNCTIONING CODE** (pragmatic, reliable) → **CORRECTNESS** (compiles, type-checks, scoped tests pass deterministically) → **CODE QUALITY** (clean, no warnings) → **POLISH** (docs, formatting).

Non-negotiables: tests drive implementation; satisfy every `qualityDimensions` entry; handle `Option`/`Result` explicitly (no `unwrap()` in production). For `estimatedEffort: "high"` or `modifiesBehavior: true` tasks, note the one alternative you rejected and why. For everything else, pick and go.

**Prohibited outcomes:**

- Tests that only assert 'no panic' without verifying outcomes (merged_slots vs failed_slots membership, HEAD position)
- Mock resolvers that bypass MERGE_HEAD / HEAD inspection — the inspection logic is the contract, not the resolver itself
- Catch-all error handlers that swallow git stderr instead of formatting it into the failed_slots diagnostic
- Spawning Claude with Dangerous permission mode — Auto with Read/Edit/Bash is the ceiling
- Leaving MERGE_HEAD set after a failed resolution — every error path must reset to pre_merge_head
- Hardcoding the task-mgr binary path in any new code — current_exe() with a labeled fallback only
- Editing any tasks/*.json file directly during loop iterations

- **[1804]** Ephemeral branch per slot satisfies git worktree constraint — slot N uses `{branch}-slot-{N}`; slot 0 reuses the loop's main branch worktree.
- **[1870]** Deterministic ephemeral branch naming via `ephemeral_slot_branch(branch_name, slot)` — already implemented.

- **[1600]** `cleanup_title_artifact` is opt-in for curate ONLY — do NOT enable it for the merge resolver.
- **[1742]** Git worktrees cannot share a checked-out branch — slot 0 must be the only worktree on `branch_name`; the resolver only ever operates in slot0_path.
- **[1001 / 1025]** Do not retry the same blocked operation more than 2-3 times; if Claude can't resolve, surface as Failed and let the wave continue.
