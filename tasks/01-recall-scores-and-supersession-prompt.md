# Claude Code Agent Instructions

You are an autonomous coding agent implementing **Recall Score Output + Learning Supersession** for **task-mgr**.

## Problem Statement

Two capabilities are missing from task-mgr's learnings system:

1. **Recall discards scores**: The recall pipeline computes rich numeric scores (FTS5 BM25, pattern-match points, vector cosine similarity, UCB bandit) but strips them before output at `src/learnings/recall/mod.rs:115`. `--format json` only exposes categorical `confidence` (high/medium/low). Consumers need parseable numeric signal.

2. **No supersession tracking**: When a learning is replaced by a better one, `invalidate-learning` destroys the old learning without linking to its replacement. A `learning_supersessions` join table enables recall to auto-filter superseded learnings and answer "what replaced this?".

## Data Flow Contracts

### Score preservation path (Feature 1)

```
Backend.retrieve(conn, &query)      -> Vec<ScoredLearning { learning: Learning, relevance_score: f64, match_reason: Option<String> }>
CompositeBackend.retrieve()         -> dedup by id (keep highest relevance_score), concat match_reasons, sort desc
recall_learnings_scored()           -> map to ScoredLearningOutput { learning, relevance_score, ucb_score: Option<f64>, combined_score: f64, match_reason }
RecallCmdResult::from_scored()      -> map to LearningSummary { ..., relevance_score, ucb_score, combined_score, match_reason }
```

Key type transitions: `ScoredLearning.relevance_score: f64` -> `ScoredLearningOutput.relevance_score: f64` -> `LearningSummary.relevance_score: f64` (all f64, no type change).

UCB is computed in `rerank_with_ucb()` inside a sort closure at `recall/mod.rs:254-279`. To capture the value, extract UCB computation to a helper (`compute_ucb_for_learning(conn, learning_id, confidence, total_window_shows) -> f64`) and call it before the sort.

### Supersession filter path (Feature 2)

```sql
-- Added to every retrieval SQL WHERE clause (alongside existing `retired_at IS NULL`):
AND l.id NOT IN (SELECT old_learning_id FROM learning_supersessions)

-- Bypassed when include_superseded = true:
-- (omit the above clause)
```

For VectorBackend (which loads embeddings directly, not through a learning JOIN):
```rust
let superseded_ids: HashSet<i64> = load_superseded_ids(conn)?;
scored.retain(|(id, _)| !superseded_ids.contains(id));
```
