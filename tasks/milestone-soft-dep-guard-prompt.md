# Claude Code Agent Instructions

You are an autonomous coding agent implementing **Milestone Soft-Dep Guard** for **task-mgr**.

## Problem Statement

A parallel-slot run wasted ~75 minutes on `cbd7d081-MILESTONE-FINAL` even though sibling tasks `REFACTOR-N-001` (in_progress) and `REFACTOR-N-002` (todo) were still active. The slot self-detected the miss via the milestone's acceptance criteria text (`"Any spawned CODE-FIX/WIRE-FIX/IMPL-FIX/REFACTOR-N tasks have passes=true"`) and emitted `<promise>BLOCKED</promise>`, but only after the wave was already sunk; the loop aborted three stale iterations later.

Root cause: the spawning task `REFACTOR-REVIEW-FINAL` created the REFACTOR-N children **without** `--depended-on-by cbd7d081-MILESTONE-FINAL`, so no `task_relationships` row links them. `select_parallel_group` saw all formal `dependsOn` deps satisfied and dispatched the milestone.

This work delivers a two-layer fix:

- **(A) Prompt-side guarantee** (FEAT-003) — teach the loop agent to pass `--depended-on-by <milestone-id>` when spawning a fix in response to a milestone's AC. Durable root-cause fix.
- **(B-narrow) Selection-side guard** (FEAT-001 + FEAT-002) — even when (A) is forgotten, defer milestone candidates whose AC mentions a known spawned-fixup prefix while a same-prefix active sibling exists. Token-aware exact-prefix matching, no regex. Milestones only — sibling fixups remain co-schedulable.

**`SPAWNED_FIXUP_PREFIXES` exact-prefix match for active IDs:**

When checking if an active id matches a fixup prefix, the id may be like `PRD-A-REFACTOR-N-001`. The simplest correct check (no `task_prefix` stripping needed):

```rust
fn id_body_starts_with_prefix_dash(id: &str, prefix: &str) -> bool {
    // Match anywhere a `{prefix}-` boundary appears, after either the start
    // OR a `-` (so `PRD-A-REFACTOR-N-001` matches `REFACTOR-N`).
    let needle = format!("{prefix}-");
    id.starts_with(&needle) || id.contains(&format!("-{needle}"))
}
```

This avoids loose `id.contains(prefix)` (which false-matches `CODE-FIXTURE-1` on `CODE-FIX`) — the trailing `-` is mandatory.
