# Claude Code Agent Instructions

You are an autonomous coding agent implementing **Dedup Dismissal Memory** for **task-mgr**.

## Problem Statement

`curate dedup` uses embedding cosine-similarity to pre-cluster learnings, then sends each cluster to the LLM to judge duplicates. When the LLM returns `[]` (no duplicates found), those same learnings get re-clustered and re-compared on every subsequent run because embeddings don't change. This wastes LLM calls and user time reviewing identical "no duplicates" output.

The fix: persist pairs the LLM has already examined and found distinct in a `dedup_dismissals` table, and skip clusters where all pairs are already dismissed on future runs.

## Data Flow Contracts

### CLI → DedupParams → curate_dedup

```
CLI flag --reset-dismissals (bool)
  → src/cli/commands.rs :: CurateAction::Dedup { reset_dismissals }
  → src/main.rs :: CurateAction::Dedup handler
  → src/commands/curate/types.rs :: DedupParams { reset_dismissals: bool }
  → src/commands/curate/mod.rs :: curate_dedup(conn, params)
```

### BatchOutput → batch_item_ids alignment

```
batches: Vec<Vec<DeduplicateLearningItem>>  // consumed by process_batches_parallel
batch_item_ids: Vec<Vec<i64>>               // saved BEFORE consumption, indexed by position
  ↕ aligned by index position
BatchOutput { batch_idx: usize, ... }       // batch_idx indexes into batch_item_ids
```

### dedup_dismissals table → HashSet<(i64, i64)>

```
SQL: SELECT id_lo, id_hi FROM dedup_dismissals
  → HashSet<(i64, i64)> where invariant id_lo < id_hi always holds
  → is_fully_dismissed checks all C(N,2) pairs against this set
```
