# Model Selection Redesign — Provider-First Config, Capability Tiers, Multi-Provider Orchestration

Condensed architecture record (FR-001 / FR-010). See `src/loop_engine/model.rs`, `project_config.rs`, and the MODELS block generated by `gen-docs` for the live details.

## Tiers

Provider-neutral `CapabilityTier` (Ord, no legacy aliases):

- cheapest
- cost-efficient
- standard
- frontier

`CapabilityTier::ALL`, `as_str`/`parse` (strict; CONFIG ERROR on typos or `opus`/`sonnet` etc.), `index`/`from_index`.

Tier membership is **config exact-match** in a provider's `tiers` map (after `[1m]` strip in `tier_of`). Substring classification is structurally dead (fable-5 contains no "opus").

## Anchor Window + Difficulty Offset

Single normalizer (`normalize_difficulty` + `difficulty_rank` reused everywhere; no second table).

`difficulty_offset`: low → -1, medium → 0, high → +1, absent/unknown → 0.

`anchored_tier(anchor, difficulty)`: `anchor.index() + offset`, clamped to [0,3]. Never wraps.

Defaults: `anchor=standard` → low=cost-efficient (sonnet), med=standard (opus), high=frontier (fable).

End clamps documented and tested (cheapest+low stays cheapest; frontier+high stays frontier).

## Per-Provider Ladders + Effort

`models.providers.<name>` (claude/grok/codex):

- `enabled`
- `tiers`: sparse map tier-key → model | null (null = omit -m/--model flag)
- `effort`: difficulty → level | null
- `fallback`: different enabled provider for rung-4 tier-preserving pivot

Claude default: full 4-rung ladder (haiku/sonnet/opus/fable).

Grok/Codex default: single standard rung (grok-build; codex = no model flag). Effort for codex capped at high (policy in `CODEX_EFFORT_FOR_DIFFICULTY` and validator).

`resolve_models_config` projects validated config into `ResolvedModelsConfig` (cheap lookups: `model_for` (nearest-defined clamp, cheaper on tie), `tier_of`, `effort_for`).

`builtin_resolved_models` for recovery paths that predate threaded config.

## Precedence Rungs (resolve_execution_plan)

Highest to lowest (see `ExecutionPlan`, `PlanContext`, `finalize_plan`):

1. Explicit per-task model (from DB row or prd-level).
2. `routing.byIdPrefix` (dash-boundary after prefix strip; optional `tier` pin). Beats the built-in review-class force — pinned by `precedence_matrix::byidprefix_route_beats_review_class_force`.
3. `routing.taskClasses` (implementation/planning/review). The review class (`is_frontier_class` — built-in, strips 8-hex prefix, excludes REFACTOR-REVIEW-*) forces Frontier at this rung — not redefinable via config, never spillover-eligible. May supply `providerPreference` + `byDifficulty` + `forceTier`.
4. QUOTA_BLACKOUT reroute (only for default-path tasks; tier preserved; only if spillover-eligible).
5. ANCHOR_WINDOW (primaryProvider + anchored_tier).

After selection: model = `model_for(p, tier)`, effort = `effort_for(final_p, difficulty)` (final provider's table wins).

`primary_runner_match` and the RuntimeError cross-provider promotion arms were DELETED (REFACTOR-006, resolved migrate-vs-delete as delete): preflight hard-rejects `primaryRunner`/`fallbackRunner`, so those paths could never fire in production. A RuntimeError-crashing Claude task climbs the same-provider tier ladder then auto-blocks; cross-provider rescue exists only on the overflow rung-4 `providers.<source>.fallback` pivot.

## Blackout Channel (separate from promotions)

`provider_blackouts` (BlackoutState in IterationContext):

- Ephemeral, per-pass, not persisted, cleared on restart.
- Written only by rate-limit reaction.
- Consumed by resolver (for reroute), deferral reaction, excluded-ids.

`runner_overrides` (via `promote_once`):

- Permanent (cross at most once per run) for overflow rung-4 + RuntimeError escapes.
- Idempotency guard is centralized in `promote_once`.

**Contract**: the two channels are disjoint. Blackout reroute never writes runner_overrides (would permanently strand task on spillover). promote_once sites never touch blackouts. Guarded by `blackout_reroute_leaves_runner_overrides_untouched`.

## Codex Route-Only Pinning

Codex is a `Provider` but `provider_for_model` never returns it. Only explicit routes (`routing.byIdPrefix` or `taskClasses` with `provider:"codex"`) select it. `resolve_execution_plan` and preflight enforce this. No model-string inference anywhere.

Fallback (rung 4) for codex is its `providers.codex.fallback` (tier-preserving) or block.
